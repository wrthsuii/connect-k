### Реалізація на Haskell 

**Опис**
Функційна реалізація ігрового ШІ (алгоритм MinMax із Alpha-Beta відсіканням) та логіки гри "k-в-ряд" з використанням чистих функцій та незмінних структур даних. Ігрове поле представлено як двовимірний список `[[Int]]` (аліас `Board`). Замість імперативної мутації стану використовується генерація нових дощок на кожному кроці. Математично елегантний аналіз "вікон" через композицію функцій та сильна статична типізація.
Головна функція: `alphaBeta :: Params -> Board -> Int -> Int -> Int -> Player -> (Int, Maybe Int)`

**Особливості**
* **Незмінні структури:** Використання базових списків (List) без побічних ефектів.
* **Чисті функції:** Функція `makeMove` не змінює поточну дошку, а повертає її нову копію.
* **Функції вищого порядку:** Інтенсивне використання `map`, `filter`, `any`, `concatMap` для обходу матриці та підрахунку евристики.
* **Безпека типів (Монада Maybe):** Використовується `Maybe Int` для безпечного повернення стовпця для ходу (запобігає помилкам, якщо ходів більше немає).
* **Рекурсія замість циклів:** Алгоритм Alpha-Beta реалізовано через взаємну рекурсію допоміжних функцій `maximize` та `minimize`.
